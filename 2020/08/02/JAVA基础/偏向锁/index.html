偏向锁

锁的简单描述

在了解偏向锁之前先了解下几个锁的大致区别。

在JDK6开始，syn有了很多的优化技术，如适应性自旋（Adaptive Spinning） 、 锁消除（Lock Elimination） 、 锁粗化（LockCoarsening） 、 轻量级锁（Lightweight Locking） 和偏向锁（Biased Locking） 等， 这些技术都是为了在线程之间更高效地共享数据， 以及解决竞争问题， 从而提高程序的执行效率。

重量级锁：由于线程切换以及锁的控制需要由用户态转换到内核态（线程的中断由内核实现），涉及到线程的上下文保存和恢复，是一个高成本的操作，所以称为重量级锁。

轻量级锁：轻量级锁的本意是在没有多线程竞争的前提下，减少重量级锁使用系统互斥产生的性能消耗。主要实现是通过JAVA线程的栈上申请的Lock Record去cas记录锁对象的Mark Word来实现。

偏向锁：目的是为了消除在在无竞争的情况下的同步原语， 在轻量级锁的基础上更进一步的消除了cas操作，偏向锁会偏向第一个获取它的线程，如果没有竞争发生，则接下来该线程获取锁都不需要进行同步操作，如果发生竞争则会发生锁升级。
